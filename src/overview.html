Win-ini-lib

This project provides simple API for handling Windows .ini configuration files.

Windows .ini file format

Windows .ini files serve for saving the configuration of programs. They consist of sections, which are blocks of related properties. These properties are key-value pairs, with values of different type and arity. The values can be further restricted (e.g. it can only contain positive numbers), can contain comments, etc. For bigger overview, we recommend you visiting Wikipedia.

Project overview

The project consists of 4 packages. All of them are prefixed with cz.cuni.mff.d3s.pp.wininilib. The main package contains most important classes. These are:

  • IniFile: serves for specifying new format, loading existing configuration, accessing its internals as well as saving it back.
  • Section: represents one section and provides methods to work with its properties
  • Property: represents one property and provides method to work with its key and values
  • Value: interface for all possible property values
  • ValueRestriction: interface for Value restrictions
Package values contains several implementations of Value interface (such as value for string, float or boolean). Package values.restrictions then contains implementations of ValueRestriction interface; therefore classes, which check the validity of specific value. Finally, the last package exceptions contains all exceptions that are thrown by the library code.

Compilation

To compile the library, Java SDK >= 1.7.0. together with Apache Ant is required. Then these Ant tasks are available:

  • compile: compiles all sources (into out/classes folder )
  • refdoc: creates project documentation (in out/javadoc)
  • create-jar: creates jar file of the library (out/jars)
  • main: performs all previous tasks
